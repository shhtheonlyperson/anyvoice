# AnyVoice Design Direction

## Source

This UI follows `~/DESIGN.md`: warm cream canvas, coral primary actions, dark product surfaces, editorial serif headlines, and humanist sans controls.

## Product Read

AnyVoice should feel like a focused studio console. The first screen is the cloning workflow itself: voice source, target text, consent, runtime state, and output. No marketing preamble.

## Visual System

- Canvas: `#faf9f5`
- Soft surface: `#f5f0e8`
- Card surface: `#efe9de`
- Ink: `#141413`
- Coral CTA: `#cc785c`
- Dark runtime surface: `#181715`
- Teal status accent: `#5db8a6`
- Amber caution accent: `#e8a55a`

## Interaction Notes

- Recording and upload are equal first-class inputs.
- Consent is a hard gate, visually close to the submit action.
- The right rail is for runtime truth: model, mode, worker state, and output.
- zh-Hant is default; English is available from a compact toggle.
